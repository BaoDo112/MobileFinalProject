# Plan 02-01: Build screen architecture and navigation map

## Goal

Define the minimum screen architecture that supports all core user outcomes without navigation overload.

## Tasks

1. List all screens by role and shared shell.
2. Map entry points, primary routes, and secondary transitions.
3. Mark requirement coverage per screen.

## Verification

- No core requirement lacks a screen destination.
- Navigation remains easy to understand.
- Shared versus role-specific screens are explicit.
