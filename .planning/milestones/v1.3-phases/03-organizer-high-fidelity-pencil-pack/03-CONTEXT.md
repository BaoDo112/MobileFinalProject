# Phase 3 Context: Organizer High-Fidelity Pencil Pack

## Goal

Extend the Organizer journey in `project.pen` from one reference screen to a complete operational high-fidelity screen set.

## Inputs

- Phase 1 route-to-screen matrix and design contract
- existing high-fidelity Organizer Tools sample frame
- archived v1.1 mid-fi Organizer board for flow coverage
- implemented Organizer screens in `apps/mobile/src/screens/`

## Locked Decisions

- Organizer visuals should remain part of the same brand system but can lean more operational and structured.
- The existing Organizer Tools frame is the base reference, not a throwaway draft.
- Dashboard, form building, submission pipeline, and submission review must be explicit enough for later backend review.

## Expected Output

- High-fidelity Organizer pack for dashboard, exhibition studio, form builder, submission pipeline, submission review, and profile
- Consistent operational pattern library across list, editor, and review surfaces
