# Plan 01-01: Create the Route-to-Screen Matrix and Shared Design Contract

## Outcome

Lock the screen inventory and reusable design system for the full high-fidelity Pen milestone before any new high-fidelity frames are added.

## Tasks

1. Enumerate the implemented route surface from the mobile navigator and supporting screens.
2. Compare that surface to the current Pen file and identify missing high-fidelity screens.
3. Decide which existing high-fidelity frames are canonical, which need refinement, and which new screens must be added.
4. Record the shared design contract that phases 2 and 3 will use.

## Validation Target

- Every implemented route has a named Pen counterpart target.
- Shared design rules are explicit enough to prevent visual drift in later phases.
