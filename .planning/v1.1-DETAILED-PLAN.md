# v1.1 Detailed Planning Blueprint

## Objective

Lock full project scope and produce complete mid-fidelity wireframe references in Pencil before implementation resumes.

## Phase Plan

## Phase 1 - Requirement Decomposition and Acceptance Matrix

### Outputs

- Master requirement matrix with IDs, acceptance criteria, and edge states.
- Role split: Visitor vs Organizer responsibility map.
- Out-of-scope guardrails for milestone v1.1.

### Done Criteria

- Every active requirement has measurable acceptance criteria.
- Every requirement has at least one mapped screen intent.
- Conflict list (ambiguous or overlapping requirements) is resolved.

## Phase 2 - Information Architecture and End-to-End Flow Map

### Outputs

- IA sitemap for all primary screens and secondary states.
- Visitor E2E flow map and Organizer E2E flow map.
- State matrix for loading, empty, error, validation, success.

### Done Criteria

- All flows are connected without dead ends.
- Every major transition is mapped to trigger/condition.
- Flow matrix references requirement IDs.

## Phase 3 - Visitor Mid-Fidelity Wireframe Pack

### Outputs

- Visitor wireframes in Pencil for:
  - Login/role entry
  - Gallery home and filtering
  - Gallery detail
  - Event registration
  - Passport vault
  - Rating and comment states

### Done Criteria

- English copy is consistent across all visitor screens.
- All primary/secondary actions are visible and labeled.
- State variants are represented (happy path + fallback states).

## Phase 4 - Organizer Mid-Fidelity Wireframe Pack

### Outputs

- Organizer wireframes in Pencil for:
  - Organizer dashboard
  - Create/edit exhibition
  - Dynamic form builder
  - Submission review and status updates

### Done Criteria

- Form builder states include field add/edit/remove behaviors.
- Submission review includes key status states.
- Wireframes align to requirement IDs ORG-01, ORG-02, ORG-03.

## Phase 5 - Cross-Flow QA and Design-to-API Blueprint

### Outputs

- Cross-flow QA checklist with resolved issues.
- Design-to-API mapping table:
  - Screen/state
  - Endpoint
  - Input fields
  - Output fields
  - Error handling rule
- Handoff notes for mobile/backend implementation start.

### Done Criteria

- No unresolved core-flow gaps.
- Every core screen state has a data contract mapping.
- Implementation entrypoint tasks are ready for execution milestone.

## Final Handoff Artifacts

- `.planning/ROADMAP.md` (v1.1 phase structure)
- `.planning/REQUIREMENTS.md` (updated traceability)
- `.planning/v1.0-MILESTONE-AUDIT.md` (gap baseline)
- `project.pen` (v1.1 mid-fidelity wireframe board)
